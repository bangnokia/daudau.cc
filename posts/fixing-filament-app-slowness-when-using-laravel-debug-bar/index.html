Fixing Filament app slowness when using Laravel Debugbar

August 18, 2023

By default, Laravel Debugbar enables logging views, which consumes a significant amount of resources in the Filament app. Displaying all view names can potentially cause the Debugbar to crash itself, rather than the application. Therefore, it is advisable to disable this feature, unless you are specifically working on a custom field.

Publishing the Laravel Debugbar Configuration File

To begin, execute the following command to publish the Laravel Debugbar config file:

php artisan vendor:publish --provider="Barryvdh\Debugbar\ServiceProvider"

Next, navigate to the config/debugbar.php file and disable views logging by making the following adjustment:

// ...
'views' => false,
// ...

That concludes the process. Enjoy a smoother development experience with Filament.
